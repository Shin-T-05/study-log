1．daily_retail_dataテーブルを使用して、各業種（business_category）の以下の指標を計算してください

解答
SELECT 
business_category,
ROUND(SUM(daily_revenue))AS total_revenue,
AVG(daily_revenue)AS avg_revenue,
MAX(daily_revenue)AS max_revenue,
MIN(daily_revenue)AS min_revenue
FROM daily_retail_data
GROUP BY business_category
ORDER BY total_revenue;

↓
② カテゴリごとに分ける
↓

2．daily_retail_dataテーブルから、2024年の月次売上トレンドを分析してください。以下の指標を計算してください

解答
WITH
  monthly_sales AS (
    SELECT
      STRFTIME ('%Y-%m', transaction_date) AS year_month,
      SUM(daily_revenue) AS total_revenue,
      ROUND(AVG(daily_revenue)) AS avg_daily_revenue
    FROM
      daily_retail_data
    GROUP BY
      year_month
  )
SELECT
  year_month,
  total_revenue,
  avg_daily_revenue,
  ROUND(
    (
      total_revenue - lag(total_revenue) OVER (
        ORDER BY
          year_month
      )
    ) * 100.0 / lag(total_revenue) OVER (
      ORDER BY
        year_month
    ),
    1
  ) AS growth_rate
FROM
  monthly_sales
ORDER BY
  year_month;
  
解法
① 日付 → 月に変換
↓
② 月ごとに売上集計
↓
③ 前月の売上を取得（LAG）
↓
④ 前月比（成長率）を計算
↓
⑤ 月順に並べて表示


3．daily_retail_dataテーブルを使用して、各業種の売上構成比を計算してください。以下の指標を含めてください

解答
WITH category_sales AS (
  SELECT
    business_category,
    SUM(daily_revenue) AS total_revenue
  FROM daily_retail_data
  GROUP BY business_category
)

SELECT
  business_category,
  total_revenue,
  total_revenue * 100.0 / SUM(total_revenue) OVER () AS revenue_share,
  SUM(total_revenue) OVER (ORDER BY total_revenue DESC)
    * 100.0 / SUM(total_revenue) OVER () AS cumulative_share
FROM category_sales
ORDER BY total_revenue DESC;

↓
④ 累積比率を算出
↓
⑤ 売上順に並べる


4．daily_retail_dataテーブルから、各業種の月次売上推移を分析してください。以下の指標を計算してください

解答
WITH 
  category_sales AS (
    SELECT
      business_category,
      STRFTIME('%Y-%m', transaction_date) AS year_month,
      SUM(daily_revenue) AS monthly_revenue
    FROM daily_retail_data
    GROUP BY business_category,year_month
)

SELECT 
    business_category,
    year_month,
    monthly_revenue,
    monthly_revenue * 100.0 / SUM(monthly_revenue) OVER (PARTITION BY year_month) AS revenue_share,
    monthly_revenue - LAG(monthly_revenue) OVER (PARTITION BY business_category ORDER BY year_month) AS revenue_increase
FROM category_sales
ORDER BY business_category,year_month;

解法
① 日付を月単位に変換
↓
② カテゴリ×月で売上を集計
↓
③ 月ごとの構成比を計算
↓
④ カテゴリごとに前月差分を計算


5．daily_retail_dataテーブルから、曜日ごとの売上パターンを分析してください。以下の指標を計算してください

解答
SELECT
  CASE STRFTIME('%w', transaction_date)
    WHEN '0' THEN '日曜日'
    WHEN '1' THEN '月曜日'
    WHEN '2' THEN '火曜日'
    WHEN '3' THEN '水曜日'
    WHEN '4' THEN '木曜日'
    WHEN '5' THEN '金曜日'
    WHEN '6' THEN '土曜日'
  END AS day_of_week,
  ROUND(AVG(daily_revenue)) AS avg_revenue,
  SUM(daily_revenue) * 100.0 
  / SUM(SUM(daily_revenue)) OVER () AS revenue_share,
  COUNT(*) AS transaction_count
FROM daily_retail_data
GROUP BY day_of_week
ORDER BY avg_revenue DESC;

解法
① 日付 → 曜日に変換
↓
② 曜日ごとにグループ化
↓
③ 各曜日で
   ・平均
   ・合計
   ・件数 を計算
↓
④ 構成比を算出
↓
⑤ 平均売上で並べる


6．daily_retail_dataテーブルの売上を以下の区分で分類し、分析してください

10万円以上：高額
5万円以上10万円未満：中額
5万円未満：少額

各区分について以下の指標を計算してください

件数
総売上
平均売上

結果は区分の降順で表示してください。

解答
SELECT
    revenue_category,
    COUNT(*) AS transaction_count,
    SUM(daily_revenue) AS total_revenue,
    ROUND(AVG(daily_revenue)) AS avg_revenue
FROM (
    SELECT
        CASE 
            WHEN daily_revenue >= 100000 THEN '高額'
            WHEN daily_revenue >= 50000 THEN '中額'
            ELSE '少額'
        END AS revenue_category,
        CASE 
            WHEN daily_revenue >= 100000 THEN 1
            WHEN daily_revenue >= 50000 THEN 2
            ELSE 3
        END AS sort_order,
        daily_revenue
    FROM daily_retail_data
) t
GROUP BY revenue_category, sort_order
ORDER BY sort_order;

解法
① 売上を「高額・中額・少額」に分類
↓
② 並び順用の数値を付与
↓
③ カテゴリごとに
   ・件数
   ・合計
   ・平均 を集計
↓
④ 意図した順番で表示

9．daily_retail_dataテーブルを使用して、業種ごとの7日間移動平均売上を計算してください。以下の指標を含めてください

解答
WITH base AS (
  SELECT
    business_category,
    transaction_date,
    daily_revenue,
    AVG(daily_revenue) OVER (
      PARTITION BY business_category
      ORDER BY transaction_date
      ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) AS moving_avg
  FROM daily_retail_data
)
SELECT
  business_category,
  transaction_date,
  daily_revenue,
  ROUND(moving_avg) AS moving_avg_7d,
  ROUND((daily_revenue - moving_avg) * 100.0 / moving_avg, 1) AS deviation_rate
FROM base
ORDER BY business_category, transaction_date;

解法
① カテゴリごとに日付順に並べる
↓
② 直近7日間の平均を計算
↓
③ 実績との差を出す
↓
④ ％に変換（乖離率）
↓
⑤ 時系列で表示


10．daily_retail_dataテーブルを使用して、各業種の売上変動を分析してください。以下の指標を計算してください

解答
SELECT
    business_category,
    ROUND(AVG(daily_revenue)) AS avg_revenue,
    ROUND(stddev(daily_revenue)) AS std_dev,
    MAX(daily_revenue)- MIN(daily_revenue) AS revenue_range,
    ROUND(STDDEV(daily_revenue) * 1.0 
    / NULLIF(AVG(daily_revenue), 0)
    , 2) AS cv
FROM daily_retail_data
GROUP BY business_category
ORDER BY cv DESC;

解法
① カテゴリごとにデータをまとめる
↓
② 平均を計算（基準）
↓
③ 標準偏差を計算（ばらつき）
↓
④ 最大−最小で幅を見る
↓
⑤ CVで相対的な不安定さを算出
↓
⑥ 不安定な順に並べる
